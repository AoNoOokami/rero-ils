How to use this demo instance?

Librarian account

A librarian account (cataloguing role) is available with the following credentials:

  • email: librarian@rero.ch
  • {{ _('password') }}: librarian

Editing records

Once you are identified, a menu is made visible. Through it, you can access the editor, and start cataloguing a new record.

A very simple import function is available, but only using the ISBN field. Once a correct ISBN-13 is entered into that field, hitting the {{ _('import') }} button will send a request to the BnF's SRU service.

When you display the detailed view of a record, you should see two buttons, one to {{ _('edit') }} the record, and the second to {{ _('delete') }} it.

Searching the catalog

The syntax to write complex requests in the search input is discribed on the lucene documentation.

With this syntax, the search engine (ElasticSearch) can retrieve information from all the fields of the bibliographic and items records. To see all the available fields and their structure, you can consult:

  • the JSON export available at the bottom of the detailed view of any record
  • the JSON schema of the bibliographic record and of the item record.

Reaching the dev team

You can open issues on GitHub if you want to report errors or suggest improvements.

If you prefer a less formal way, there's a chatroom on Gitter, which you can access with a GitHub account.

Comment utiliser cette installation de démonstration?

Compte bibliothécaire

Un compte bibliothécaire (avec le rôle de catalogueur) est disponible, avec les identifiants suiants:

  • email: librarian@rero.ch
  • {{ _('password') }}: librarian

Modifier les notices

Une fois que vous êtes identifié·e, un menu s'affiche. Il donne accès à l'éditeur qui permet de cataloguer une nouveau document.

Une fonction d'importation très simple est disponible, limitée au champ de l'ISBN. Lorsqu'un ISBN-13 correct est saisi dans ce champ, le bouton {{ _('import') }} envoie une requête au service SRU de la BnF.

Depuis la vue détaillée d'une notice, deux boutons sont visibles, le premier pour {{ _('edit') }} la notice, et le deuxième pour la {{ _('delete') }}.

Rechercher dans le catalogue

La syntaxe pour construire des requêtes complexes dans le champ de recherche se trouve dans la documentation de lucene.

Au moyen de cette syntaxe, le moteur de recherche (ElasticSearch) est en mesure d'accéder aux données contenues dans l'ensemble des champs des notices bibliographiques et des exemplaires. Pour connaître les champs existants et leur structure, vous pouvez consulter:

  • l'export JSON accessible au bas de chaque notice détaillée;
  • les JSON schemas de la notice bibliographique et de l'exemplaire.

Communiquer avec l'équipe de développement

Vous pouvez ouvrir des issues sur GitHub, afin de signaler des erreurs ou de suggérer des améliorations.

Si vous préférez une voie moins formelle, un salon de discussion est disponible sur Gitter, auquel vous pouvez accéder au moyen d'un compte GitHub.

Wie wird diese Demo-Instanz verwendet?

Bibliothekarkonto

Ein Bibliothekarkonto (Katalogisierungsrolle) steht mit folgenden Nutzerdaten zur Verfügung:

  • E-Mail: librarian@rero.ch
  • {{ _('password') }}: librarian

Datensatz bearbeiten

Nach dem Einloggen wird ein Menü sichtbar, über welches Sie auf den Editor zugreifen können und einen neuen Datensatz erfassen können.

Eine sehr einfache Importfunktion steht unter Verwendung des ISBN-Feldes zur Verfügung. Sobald eine korrekte ISBN-13 in das ISBN-Feld eingegeben wurde, wird durch den Klick auf {{ _('import') }} eine Anfrage an den SRU-Service der BnF gesendet.

Bei der Detailanzeige eines Datensatzes sehen Sie zwei Buttons, den einen um den Datensatz zu {{ ('edit') }} und den anderen um ihn zu {{ ('delete') }}.

Im Katalog suchen

Die Syntax, um komplexe Abfragen im Suchfeld zu schreiben, wird in der Lucene-Dokumentation beschrieben.

Mit dieser Syntax kann die Suchmaschine (ElasticSearch) Informationen aus allen Feldern der bibliographischen und der Exemplar-Datensätze finden. Um sämtliche verfügbare Felder zu sehen, können Sie:

  • den JSON-Export, verfügbar unterhalb der Detailanzeige jedes Datensatzes, anschauen;
  • das JSON-Schema eines bibliographischen Datensatzes und eines Exemplar-Datensatzes konsultieren.

Das Entwicklerteam kontaktieren

Auf GitHub können Sie ein neues Issue öffnen, wenn Sie einen Fehler melden oder eine Verbesserung vorschlagen möchten.

Wenn Sie ein weniger formales Verfahren bevorzugen, gibt es ein Chatroom auf Gitter. Das Chatroom können Sie mit einem GitHubkonto beitreten.
